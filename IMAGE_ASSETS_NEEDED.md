## Instructions for Adding Images

1. For each image, use Google Images, Bing Images, or product manufacturer websites to find high-quality product/category images
2. Download images in PNG format (preferred) or conversion tools can be used for JPG to PNG
3. Place each image in the appropriate folder:
   - Hero images → `/public/images/hero/`
   - Category images → `/public/images/categories/`
   - Product images → `/public/images/products/`
4. Use the exact filenames listed above (case-sensitive on Linux servers, but lowercase recommended)
5. Recommended image sizes:
   - Hero: 1200x1200px or 1200x600px
   - Categories: 600x600px or 600x400px
   - Products: 600x600px or 400x400px

## Fallback Behavior

If an image is missing or cannot be found:
- The app will automatically display `/public/placeholder.png` as a fallback
- This keeps the layout intact and prevents broken images
- Users will still see the product/category information even if the image is missing

## Notes

- The refactored system no longer generates SVG illustrations
- All images must now be real product/category photos from the internet or your own photography
- The system is designed to be flexible - you can update images anytime by replacing files in the folders
- No code changes are needed once images are placed in the correct folders - they will be automatically loaded
